## File Summary

### 1. Core GUI: Anchor Container
**`interface/countrypoliticsview.gui`** (lines ~76-80)
- Contains the `coalition_pol_party_bg_anchor` container that anchors the coalition highlight backgrounds behind the party grid. It's positioned inside `chart_explanation` at position `{x=5, y=3}`.

### 2. Core GUI: Highlight Sprite Templates
**`interface/kaiserreich/coalition_pol_party_bg.gui`**
- Defines 10 `iconType` elements, one per ideology (totalist through natpop), each named `coalition_pol_party_bg_<ideology>`. Each uses the sprite `GFX_coalition_pol_party_bg` and is positioned at a unique y-offset corresponding to the ideology's row in the party grid. All have `alwaystransparent = yes`.

### 3. Core GUI: Party Popularity Row
**`interface/kaiserreich/party_popularity.gui`**
- Defines the `party_popularity_number_container` with the `party_pop_text` textbox (shows `[Root.GetPopularityPercent]`) and the `popularity_ideology_icon` (shows `[GetPopIcon]`).

### 4. GFX: Coalition Background Texture
**`interface/kaiserreich/countrypoliticsview.gfx`** (lines 3-8)
- Defines `GFX_coalition_pol_party_bg` with `noOfFrames = 3` from texture `gfx/interface/coalition_pol_party_bg.png`.
  - Frame 0: No coalition role (hidden/neutral)
  - Frame 1: Previously in coalition / reset
  - Frame 2: Coalition partner (highlight)
  - Frame 3: Ruling party (highlight)

### 5. GFX: Small Ideology Icons
**`interface/kaiserreich/party_popularity.gfx`**
- Defines 10 small ideology sprites (`GFX_totalist_small`, `GFX_syndicalist_small`, etc.) from `gfx/interface/ideologies/*_small.png`.

### 6. GFX: Texture File
**`gfx/interface/coalition_pol_party_bg.png`**
- The 3-frame sprite sheet for the coalition background highlight.

### 7. Scripted GUI: Coalition Indicator
**`common/scripted_guis/00_political_parties.txt`** (lines 44-62)
- Defines `coalition_indicator` which attaches `coalition_pol_party_bg` to `coalition_pol_party_bg_anchor`. It sets each ideology's icon frame to `coalitions_array^i`:
  ```
  coalition_pol_party_bg_tot = { frame = coalitions_array^0 }
  coalition_pol_party_bg_synd = { frame = coalitions_array^1 }
  ... (through index 9 for natpop)
  ```

### 8. Scripted GUI: Party Popularity Row
**`common/scripted_guis/00_political_parties.txt`** (lines 1-15)
- Defines `party_popularity_number` which attaches `party_popularity_number_container` to `top_bar`, setting the ideology icon via `[GetPopIcon]`.

### 9. Scripted Effects: Coalition Logic
**`common/scripted_effects/00_party_popularity_effects.txt`**
- `create_coalitions_array_if_missing`: Initializes `coalitions_array` to size 10, sets ruling party index to 3.
- `add_to_coalition`: Sets `coalitions_array^index = 2`, adds coalition PP gain via `coalition_pp_gain`, adds the `party_popularity_dynamic_modifier_coalition`.
- `remove_from_coalition`: Sets `coalitions_array^index = 0`, subtracts coalition PP gain, removes the coalition dynamic modifier if no partners remain.
- `end_coalition`: Resets all coalition entries to 1, clears `coalition_stability_array` and `coalition_pp_gain`, removes the coalition dynamic modifier.
- `default_party_popularity_PP_gain`: Resets to standard ruling-party PP gain.
- `map_party_popularity_PP_gain_to_ideology`: Maps PP gain to a specific ideology index.
- `disable_party_popularity_PP_gain`: Disables all PP from party popularity.
- `change_ruling_party_popularity`: Changes ruling party popularity.
- `set_coalition_stability_<ideology>`: 10 effects setting `coalition_stability_array^i` for per-ideology stability multiplier.
- `save_current_popularities` / `save_current_coalitions`: Persistence helpers.

### 10. Scripted Triggers: Government & Coalition Checks

- `has_coalition`: Checks for `party_popularity_dynamic_modifier_coalition`.
- `is_in_coalition_with_<ideology>`: 10 triggers checking `coalitions_array^i = 2`.

## How It Works End to End

1. `coalitions_array` is a 10-entry per-country array. Each index maps to an ideology (0=totalist ... 9=natpop). Values: 0/1=not in coalition, 2=coalition partner, 3=ruling party.

2. The `coalition_indicator` scripted GUI reads `coalitions_array^i` for each ideology and sets the frame of the corresponding `coalition_pol_party_bg_<ideology>` icon. This renders a highlight behind that party's row in the politics view.

3. The highlight texture `coalition_pol_party_bg.png` has 3 frames, so frame 2 shows a "coalition partner" highlight and frame 3 shows a "ruling party" highlight. Parties with frame 0 or 1 show no highlight.

4. Coalition membership is managed through the scripted effects `add_to_coalition`, `remove_from_coalition`, and `end_coalition`. These also compute the dynamic political power contribution from coalition partners using `coalition_stability_array` multipliers.

## How It Works

**`coalitions_array`** is a per-country array (size = 7, matching the ideology count). Each index stores the party's government role:
- **0/1** = not in government
- **2** = coalition partner → renders frame 2 (coalition highlight)
- **3** = ruling party → renders frame 3 (ruling party highlight)

On game start (`on_startup`), every country's array is sized to `global.ideology_groups^num` (7), the ruling party index is set to 3, and the `party_popularity_dynamic_modifier` is added.
